# Data and provenance layout

Frozen evidence directories are immutable inputs to the portable replay. Their date suffixes identify the evaluated bundle; they are not timestamps generated by the reviewer run.

- `final_20260716`: primary cross-tool, mapping, energy, transport, and MNIST functional summaries.
- `reviewer_extension_20260717`: independent NoC oracle, hold-out, scalability, trace overhead, and controlled intervention summaries.
- `trace_visualization_20260717`: reduced spatial and temporal trace products plus their rendering/provenance manifests.
- `optical_intervention_20260717`: matched electrical/optical intervention configs, raw terminal JSON, and summary table.
- `mnist_pe_precision_20260717`: four-profile numerical and system-cost terminal summaries.

Large raw candidate directories, external tool installations, redundant checkpoints, failed-attempt workspaces, and the 3.92-GiB full trace are intentionally omitted. The package retains the terminal tables used in the paper, the selected raw optical records, configuration plans, output manifests, canonical hashes, and code needed to inspect how each reduction was formed.

Some copied records originally contained machine-local absolute paths or host labels. In this anonymous copy those identity-bearing strings are replaced with `ARTIFACT_ROOT`, `EXTERNAL_TOOL_ROOT`, or `anonymous-host`. Numerical values, candidate IDs, content hashes, and relative evidence paths are unchanged. `metadata/artifact_manifest.json` authenticates this sanitized review copy.

Fresh runs must use a new directory under `output/` or a reviewer-chosen external path. Do not write into the frozen directories, because that would mix review-time results with submission evidence.
